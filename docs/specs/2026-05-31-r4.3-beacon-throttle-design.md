# R4.3 — adaptive beacon throttle + silence-jitter (the first draw-adding slice) — design spec

**Status:** DESIGN (grounded against the Lua beacon_fire; awaiting user pins — this is the determinism-critical
checkpoint). **Date:** 2026-05-31. **Scope (user pick):** R4.3 — the thundering-herd beacon plane: a preamble
witness, the quiet-threshold gate, the silence-jitter deferred TX, the max-idle B+C override, the budget-aware
beacon skip, AND the deferred **R4.2 #00** (the min-interval second draw in `schedule_triggered_beacon`).
**THE FIRST slice that adds a new mt19937 draw** — so it is determinism-isolated and gated behind a rand-order
golden. Grounding: do this after all the draw-free slices (now done). **Source of truth:** `dv_dual_sf.lua`
(behaviour). Delicate firmware (node.cpp beacon timer + node_beacon.cpp) → implement INLINE.

---

## 1. THE DETERMINISM CONTRACT (the crux — read first)
The shared `_sim_rng` is consumed draw-for-draw by both engines. R4.3 adds new draw SITES; each must fire at the
EXACT Lua call-site order AND only under the EXACT Lua condition, or every downstream draw (RTS retry jitter,
cascade jitter, the rerank triggered-beacon, join backoff) desyncs.

**Draw sites in `beacon_fire` (Lua dv:7695-7864), in order:**
1. **silence-jitter** `rand(0, beacon_silence_jitter_ms+1)` (dv:7795-7796) — fires ONLY when **all** of:
   `quiet_threshold_ms > 0` (dv:7704 — else the fast path bypasses it) AND the gate passed (`since_rx >=
   quiet_threshold_ms` OR `force_idle`, dv:7785) AND `beacon_silence_jitter_ms > 0` (dv:7795). Otherwise NO draw.
2. **periodic re-arm** `rand(lo, hi+1)` (dv:7864) — ALWAYS, at the end (already in C++ node.cpp:70).

**Site in `schedule_triggered_beacon` (Lua dv:7877-7906) — the R4.2 #00 port:**
3. **trigger-jitter** `rand(jitter_min, jitter_max+1)` — ALWAYS (already in C++ node_beacon.cpp:230).
4. **min-interval second draw** `rand(jitter_min, jitter_max+1)` (dv:7896) — fires ONLY when **all** of:
   `steady_state` (= `!in_discovery() && now - discovery_started_ms >= beacon_boot_grace_ms(120000)`) AND
   `beacon_trigger_min_interval_ms > 0` AND `_last_beacon_tx_ms` set AND `now + delay < _last_beacon_tx_ms +
   beacon_trigger_min_interval_ms`. Otherwise NO draw.

**THE INVARIANT THAT KEEPS EVERY EXISTING GATE BYTE-IDENTICAL:** all current gates set `quiet_threshold_ms=0` →
the `quiet_threshold_ms <= 0` fast path (dv:7704-7708) → **NO silence-jitter draw** → only the periodic re-arm
(unchanged). And every gate runs UNDER `beacon_boot_grace_ms`=120000 → `steady_state` false → **NO min-interval
draw**. So R4.3 is byte-identical on all 6 current gates. The new draws fire only in a scenario with
`quiet_threshold_ms>0` (both engines) or `sim_time>120000` (both engines). **Lock with rand-order unit tests; the
gate-set stays band-0; ONLY after R4.3 may a `>120000ms` draw-for-draw differential exist.**

## 2. Mechanism (Lua beacon_fire dv:7695-7864)
- **Witness** (`_last_rx_routing_sf_ms`): set on EVERY routing-SF decode (on_recv top, dv:9160-9164) AND on
  `on_preamble_detected` (dv:12224-12226 — the SX1262 IRQ; currently a NO-OP stub at node.cpp). `_last_rx_bcn_ms`:
  set on beacon ingest (dv:9552-9559). Pure timestamp writes — NO rand.
- **`beacon_fire` body** (replaces `emit_beacon("periodic")` in `on_timer(kBeaconTimerId)`):
  - join-not-ready / busy-in-data-exchange (`_pending_tx||_pending_rx`) → skip (no beacon, no draw).
  - `quiet_threshold_ms <= 0` → **fast path:** `emit_beacon("periodic")` (no throttle, NO silence-jitter draw).
  - else: `since_rx = now - _last_rx_routing_sf_ms` (∞ if never); **max-idle B+C** (dv:7734-7784): if
    `since_tx >= beacon_max_idle_ms`, compute `dirty_n` + `since_bcn_rx`; if `dirty_n==0 && since_bcn_rx <
    max_idle/3` → `beacon_max_idle_skip_clean` (fall through, throttle will skip); else `force_idle=true` +
    `beacon_max_idle_force`. Then: if `since_rx < quiet_threshold_ms && !force_idle` → `beacon_skipped_busy`
    (stage=pre_jitter, NO draw). else → **silence-jitter draw**; `jitter==0` → emit now; `jitter>0` → defer
    `after(jitter, kBeaconJitterTimerId)`.
  - **deferred re-check** (`on_timer(kBeaconJitterTimerId)`, dv:7801-7849): busy → skip; recompute `since` +
    `force_idle_post` (same B+C); `since < quiet && !force_idle_post` → `beacon_skipped_busy` (stage=post_jitter,
    return); else `emit_beacon("periodic")`. NO draw in the deferred path.
- **budget-aware skip** (dv:7588-7604, in `send_beacon_page`): skip the beacon if `compute_budget_tier() >=
  CRITICAL` (don't spend a near-exhausted budget on a beacon). Hooks into `emit_beacon` entry.

## 3. C++ change set (INLINE)
1. `node.h` — `+uint64_t _last_rx_routing_sf_ms = 0; +uint64_t _last_rx_bcn_ms = 0;`. `NodeConfig
   +uint32_t beacon_silence_jitter_ms = 10000;` (config-overridable). `+kBeaconJitterTimerId` (next free id).
2. `node.cpp on_recv` — set `_last_rx_routing_sf_ms = now()` on a successful routing-SF decode (the dispatch top);
   `_last_rx_bcn_ms = now()` in the beacon branch. `on_preamble_detected(time_ms)` — un-stub → set
   `_last_rx_routing_sf_ms = time_ms`. (Determinism: the witness MUST be set on BOTH preamble AND decode, or the
   gate timing — and thus whether the jitter draw fires — differs between engines.)
3. `node.cpp on_timer(kBeaconTimerId)` — replace `emit_beacon("periodic")` with the `beacon_fire` body (§2);
   keep the re-arm draw. `+on_timer(kBeaconJitterTimerId)` for the deferred re-check.
4. `node_beacon.cpp emit_beacon` — budget-aware skip at entry (tier >= CRITICAL).
5. `node_beacon.cpp schedule_triggered_beacon` — add the #00 min-interval second draw (§1 site 4).
6. `FirmwareNode.cpp` — plumb `beacon_silence_jitter_ms`.

## 5. Gate / tests
- **Rand-order golden (THE critical test):** (a) `quiet_threshold_ms=0` → a periodic beacon fire draws EXACTLY one
  rand (the re-arm), NOT two — proves the fast path skips the silence-jitter draw (else every existing gate breaks).
  (b) `quiet_threshold_ms>0` + a stale `_last_rx_routing_sf_ms` (gate passes) → draws TWO (silence-jitter + re-arm).
  (c) `quiet_threshold_ms>0` + a fresh witness (gate fails) → draws ONE (re-arm only), emits `beacon_skipped_busy`.
  (d) the #00: `schedule_triggered_beacon` at `_now>120000` with recent `_last_beacon_tx_ms` → TWO draws; at
  `_now<120000` → ONE.
- **Witness unit:** `on_preamble_detected` + an on_recv decode both update `_last_rx_routing_sf_ms`.
- **Deferred path unit:** `quiet>0`, gate passes, `jitter>0` → no immediate beacon; `on_timer(kBeaconJitterTimerId)`
  with a still-stale witness → `beacon_tx`; with a fresh witness → `beacon_skipped_busy` (post_jitter), no beacon.
- **Differential (band-0):** all 6 existing gates stay band-0 (quiet=0 fast path, sim<120s). A NEW throttle
  scenario (`quiet_threshold_ms>0`, multi-node) is the first `>0`-band differential — calibrate from a multi-seed
  run (the silence-jitter draw means the band may be `>0`; REPORT the funnel, assert the delivery band).
