# R4.2 — persistent neighbor tier mark + route penalty + ACK budget_hint — design spec

**Status:** IMPLEMENTED + verified (103/103 native, all 5 differential gates band-0, full suite 110/110). Post-review
in §7a. **Date:** 2026-05-31. Originally DESIGN (grounded against the Lua + the current C++ scoring).
**Scope (user pick):** R4.2 — turn the short-term BUDGET-NACK blind window into routing-grade demotion. Stores the
persistent neighbor budget tier, subtracts a tier penalty from the candidate score (`effective_score`), reranks
affected routes, and adds the ACK `budget_hint` piggyback (the second writer into the tier map). **Completes the
budget plane** (R4.0 tier → R4.1 NACK blind+requeue → R4.2 routing demotion). **Source of truth:** `dv_dual_sf.lua`.
Delicate firmware (the SCORING core) → implement INLINE. USER commits.

**THE RISK (grounding flagged it):** this is the FIRST real `effective_score` consumer — today `route_strictly_better`
uses raw `score` (penalties hardcoded 0, node.h:60-61). Subtracting a penalty can FLIP route rankings and de-align
flights. Isolated from R4.1 so a ranking regression can't poison the clean NACK slice.

---

## 1. The Lua mechanism (verified)
- **Penalty table** `TIER_SCORE_PENALTY_BY_ALTS_DB[tier][viable_alts]` (dv:3843-3848), Q4 dB:
  `[0]={0,0,0} [1]={16,64,112} [2]={112,224,336} [3]={128,240,400}`. Bigger penalty when MORE viable alternatives
  exist (demote harder when there's somewhere else to go). tier 0 (HEALTHY) = no penalty.
- **`get_neighbor_tier(node_id)`** (dv:3856-3871): reads `neighbor_budget_tier[node_id]`, **TTL-expiring**
  (`neighbor_budget_tier_ttl_ms`=300000; lazy-prune on read — clears the entry + set_at when expired). 0 if no mark.
- **`viable_alternatives_for_candidate(c, candidates, viab_q4)`** (dv:3874-3884): count OTHER candidates with **RAW**
  `score >= viab_q4` (routing SNR floor), capped at 2.
- **`budget_penalty_db(c, candidates, viab_q4)`** (dv:3887-3894): tier<=HEALTHY → 0; else `table[tier][viable_alts]`.
- **`effective_score(c, candidates, viab) = c.score − budget_penalty − suspect_penalty`** (dv:4050-4053). *Suspect
  penalty is the peer-liveness plane — NOT R4.2; stays 0.*
- **`route_strictly_better(a,b,viab,candidates)`** (dv:4227-4245): computes `effective_score` for both, viability on
  the EFFECTIVE score (a penalty can drop a candidate below viable), then viable-first / hops-first / eff-score-tie.
- **`mark_neighbor_budget_tier(node_id, tier, source, local_only)`** (dv:4320-4342): skip if tier<=HEALTHY; **max-merge**
  (`if current > tier return 0` — never downgrade a worse mark); store tier + set_at(now); `resort_routes_for_neighbor_penalty`;
  emit `neighbor_budget_mark`; return reranked.
- **`resort_routes_for_neighbor_penalty(node_id, source, local_only)`** (dv:4255-4318): for each rt entry that HAS a
  candidate via node_id AND has >1 candidate → `sort_route_candidates` (now penalty-aware); if the PRIMARY changed →
  `entry.dirty=true` (unless local_only), reranked++, emit `rt_penalty_rerank`. If reranked>0 && !local_only →
  **`schedule_triggered_beacon`**.
- **ACK `budget_hint`** — emit (dv:11005-11007, 11053-11055): the ACK piggybacks `min(CRITICAL, my_budget_tier)`.
  consume (handle_ack, dv:10341-10359): on an ACK with `budget_hint > HEALTHY` →
  `mark_neighbor_budget_tier(ack_src, hint, "ack_budget", local_only=true)`. So a node learns its next-hop's tier
  from the ACK (forward direction), in addition to the NACK (reverse).

## 2. Determinism — the conditional triggered-beacon draw (the subtle part)
`schedule_triggered_beacon()` (node_beacon.cpp:223-230) DRAWS a rand — but coalesces BEFORE it
(`if (_triggered_beacon_pending) return;`). `resort_routes_for_neighbor_penalty` calls it when a rerank changed a
primary (and !local_only). So R4.2's `mark_neighbor_budget_tier` (via the NACK react, local_only=false) CAN draw —
**but ONLY when a real tier mark causes a primary-changing rerank.** In every differential GATE all nodes stay
HEALTHY (no node reaches CRITICAL — see the R4.1 gate reality), so: no NACK / no ACK-hint>0 → no mark → no resort →
no triggered beacon → **zero new draws, gate-identical.** The draw only fires when the budget plane ACTIVATES, and
then it matches the Lua draw-for-draw (same call-site, same coalesce gate). The ACK-hint writer is `local_only=true`
→ NO triggered beacon (and no dirty) on that path, so it never draws. **So: gate-draw-free; activated-path draws
faithfully; lock the activated path with a rand-order unit test.** (This corrects the grounding's "R4.2 draw-free".)

## 3. C++ change set (INLINE — the scoring core)
1. `node.h` — `std::map<uint8_t,uint8_t> _neighbor_budget_tier;` + `std::map<uint8_t,uint64_t> _neighbor_budget_tier_set_at;`.
   Decls: `uint8_t get_neighbor_tier(uint8_t) /*non-const: lazy-prune*/; int16_t budget_penalty_q4(const RtCandidate&, const RtCandidate*, uint8_t n) const; int16_t effective_score(const RtCandidate&, const RtCandidate*, uint8_t n) const; int mark_neighbor_budget_tier(uint8_t, uint8_t tier, const char* source, bool local_only); int resort_routes_for_neighbor_penalty(uint8_t, const char*, bool);`
2. `node_routing.cpp` — `route_strictly_better` + `sort_candidates` gain a **candidate-list context** (Pin P-CTX) so
   `effective_score` can count viable_alts; the comparisons switch raw `score` → `effective_score`. `rt_merge` passes
   the entry's candidate array as context.
3. `node_routing.cpp` (or node_mac) — `get_neighbor_tier` (TTL prune), `budget_penalty_q4`, `effective_score`,
   `mark_neighbor_budget_tier`, `resort_routes_for_neighbor_penalty`. Penalty table as a `constexpr int16_t[4][3]`.
4. `node_mac.cpp handle_nack` reason-1 — replace the R4.1 `reranked=0` no-op with
   `mark_neighbor_budget_tier(pt.next, tier, "nack_budget", false)` and emit the real `reranked`.
5. `node_mac.cpp handle_data` ACK emit — `ain.budget_hint = min(CRITICAL, compute_budget_tier())` (was hardcoded 0).
6. `node_mac.cpp handle_ack` — decode `k.budget_hint`; if `> HEALTHY` →
   `mark_neighbor_budget_tier(meta.src_hint, hint, "ack_budget", true)`.

## 5. Gate (same shape as R4.1 — budget plane is path-not-delivery + gate-inert when healthy)
- **Unit:** the penalty table + `get_neighbor_tier` TTL expiry (table-exact vs dv:3843, lazy-prune at the boundary);
  `effective_score` demotion (a CRITICAL primary with a viable alt → alt becomes primary after `mark_neighbor_budget_tier`);
  the `viable_alts`-indexed penalty (0/1/2 alts → different penalty); the ACK-hint writer marks with local_only (no
  rerank-beacon); a rand-order golden on the activated mark→rerank→`schedule_triggered_beacon` (draw fires once, and
  NOT when local_only / no primary change / already-pending).
- **Differential (band-0):** the `r6` HEALTHY gate already proves inertness (healthy → no mark → no rerank → no draw,
  delivery + routes identical). Optionally extend r6 to assert `rt_penalty_rerank` count == 0 on both engines.

## 7a. Post-review (R4.2 adversarial review, 2026-05-31: 27 found → 11 surviving)
**Fixed (LOW):** #05 added the `meta.src_hint >= 0` guard to the handle_ack consume gate (matches the NACK gate
+ Lua dv:10300); #07 added a unit test for the EXHAUSTED→CRITICAL ACK-hint cap (drives a mid-flight tier climb
by bumping the scripted airtime between RTS and DATA); #06/#03/#08/#10 corrected stale comments (the "2-bit can't
carry EXHAUSTED" claim, the cascade "effective_score==score" / "next_hop tie-break" comments, the RtCandidate
penalty comment). **103/103 native.**

**Deferred — both gate-inert (the budget plane never activates in a differential gate; the activated path is
unit-tested), documented as follow-ups:**
- **#00 (HIGH/PLAUSIBLE → R4.3):** `schedule_triggered_beacon` omits the Lua's CONDITIONAL second rand draw
  (the min-interval defer, dv:7896): when `steady_state && now+delay < last_beacon_tx_ms + beacon_trigger_min_interval_ms`
  the Lua draws TWICE. This is the **beacon-throttle min-interval — explicitly R4.3 scope** (R2 deferred it). The
  verifier confirmed it's UNREACHABLE in every gate (all run under `beacon_boot_grace_ms`=120000, so `steady_state`
  is never true; the one >120s scenario is the non-wired bernoulli distribution band). **R4.3 must port this and
  add a rand-order test at `_now > 120000` with a recent `_last_beacon_tx_ms` (TWO draws). GUARD: do NOT add a
  >120000ms hard draw-for-draw gate before R4.3 lands.**
- **#01 (MEDIUM/CONFIRMED → cascade-refresh follow-up):** the Lua `pick_next_cascade_hop` calls `refresh_route_order`
  FIRST (penalty-aware re-sort + maybe a triggered-beacon draw) to catch a tier change since the last sort; the C++
  reads the already-(mark-time)-sorted `_rt`. Gate-inert; the gap is the TTL-expiry-between-mark-and-cascade edge
  (mark already re-sorts immediately). Noted in node_cascade.cpp.
- **#02 (sort stability):** stable insertion sort vs Lua's unstable `table.sort` could diverge on an EXACT
  eff-score+hops+viability tie — pre-existing, gate-inert (gates use well-separated SNRs), no fix (Lua-faithful).
- **#04 (telemetry stats):** `neighbor_budget_mark` omits the Lua's per-entry stats (primary_demoted, …) — the
  deferred C++-NACK-telemetry-schema class (events not byte-for-byte; the differential keys on event NAMES).
- **#09 (viability floor):** `budget_penalty_q4`/`route_strictly_better` hardcode `_routing_snr_floor_q4`; the Lua
  threads it per call site — only matters under gateway-schedule / cross-layer SF adoption (future).
