# R4.4 — originator anti-spam (1st-hop statistical rate-limit) — design spec

**Status:** IMPLEMENTED + verified (107/107 native, all 5 gates band-0). Post-review in §7a (the P-CAP pin was
REVISED — see below). **Date:** 2026-05-31. Originally DESIGN. **Scope (user pick):** R4.4 — a self-contained
1st-hop check at the top of `handle_rts` that silently drops an RTS from a node behaving like an originator-flooder.
**Draw-free**, independent of the budget/beacon planes (only borrows R4.0's `_duty_cycle_budget_ms` for the airtime
backstop). The **last draw-free R4 slice** (grounding: land all draw-free before the draw-adding R4.3/R4.5).
**Source of truth:** `dv_dual_sf.lua`. Delicate firmware (node_mac.cpp handle_rts/handle_cts) → implement INLINE.

---

## 1. The mechanism (Lua)
Every node OVERHEARS the routing-SF broadcast traffic and keeps a per-sender sliding-window ledger of the RTSes and
CTSes each neighbour emits. A legitimate FORWARDER emits ~1 CTS per inbound flight and ~1 RTS per outbound forward,
so `R[X] ≈ C[X]`; an ORIGINATOR sends RTSes without preceding CTSes, so `R[X] − C[X]` climbs.

- **`track_originator_observation(sender, kind, ctr_lo, air)`** (dv:3205-3239): append `{t, kind, ctr_lo, air}` to
  `per_sender_originator[sender].events`; prune events older than `originator_window_ms`(300000); **dedup** — if the
  FIRST kept event with the same `kind+ctr_lo` is within `originator_retry_dedup_ms`(10000), REFRESH its `t=now` and
  DON'T append (a retry is not a new origination).
- **`compute_originator_metric(sender)`** (dv:3247-3277) → `(apparent, total_air, rts, cts)`: over the window count
  **DISTINCT `ctr_lo` per kind** (a multi-second stall that escapes the 10s track-dedup must not look like a flood —
  ctr_lo is 4-bit so this caps at 16, well above the ~6 threshold); sum `air` CUMULATIVELY (retries really burn
  channel time). `apparent = max(0, rts − cts)`.
- **Track sites:** `handle_rts` tracks `("rts", r.ctr_lo, rts_airtime)` for `meta.src` — for EVERY overheard RTS,
  BEFORE the addressed-to-us check (dv:9709-9712); `handle_cts` tracks `("cts", c.ctr_lo, cts_airtime)` for `meta.src`
  for EVERY overheard CTS (dv:10115). **Gateway cross-layer forwards (`RTS_FLAG_RELAY`) are EXEMPT** (dv:9709 `if not
  r.relay`) — not a 1st-hop origination.
- **Drop** (`handle_rts`, addressed-to-us path, BEFORE the budget NACK + CTS, dv:9990-10012): `app_orig, total_air =
  compute_originator_metric(meta.src)`; `cap = floor(originator_airtime_share(0.25) × duty_cycle_budget_ms)`; if
  `app_orig > originator_max_per_window(6)` OR `total_air > cap` → emit `rts_drop_originator_throttle`, **silent drop
  (no CTS, no NACK), return.**

## 2. Determinism + gate reality
**Draw-free** (ledger append/scan + integer threshold). The ledger MUST be insertion-ordered (a `std::vector` per
sender, NOT a map) so the dedup-FIRST refresh matches the Lua `ipairs` scan. The metric itself is order-independent
(distinct-ctr_lo set + cumulative air). **Gate reality (same family as the budget plane):** on COOPERATIVE traffic a
forwarder's `apparent ≈ 0` and `total_air ≪ cap`, so the check is INERT on both engines → band-0. The throttle
ACTIVATING (dropping a spammer) is the un-gateable direction (a silent drop, path-not-delivery; AND the airtime
backstop is wire-divergent — the C++ RTS is shorter, so `total_air` differs between engines, only matters for
high-volume spam) → **UNIT-tested.** The count-based `apparent` threshold IS wire-independent (distinct ctr_lo).

## 3. C++ change set (INLINE)
1. `node.h` — `NodeConfig.originator_max_per_window = 6` (T-class). `struct OrigEvent { uint64_t t; uint8_t kind;
   /*0=rts,1=cts*/ uint8_t ctr_lo; uint32_t air; };` + `std::map<uint8_t, std::vector<OrigEvent>> _per_sender_originator;`
   Decls: `track_originator_observation(sender, kind, ctr_lo, air)`; `compute_originator_metric(sender, &app, &air, &rts, &cts)`.
2. `node_mac.cpp` — the two functions. Prune+dedup-first in track; distinct-ctr_lo (a 16-bit seen-mask per kind, since
   ctr_lo is 4-bit → bits 0..15) + cumulative air in compute.
3. `node_mac.cpp handle_rts` — track `"rts"` at the TOP (after parse + leaf_id, BEFORE the `r.next != self` return),
   exempt `r.rts_flags & RTS_FLAG_RELAY` (relay decode: if the port doesn't expose it yet, the hook is a no-op since
   gateway relay is R7). The DROP goes in the addressed-to-us path, BEFORE the R4.1 budget NACK.
4. `node_mac.cpp handle_cts` — track `"cts"` BEFORE the `c.to != self` return.
5. `FirmwareNode.cpp` — plumb `originator_max_per_window`.

## 5. Tests
- **Unit `compute_originator_metric`:** feed a scripted RTS/CTS event sequence (same-ctr_lo retries inside the 10s
  dedup; events straddling the window prune; ctr_lo reuse across the window) → assert `(apparent, total_air, rts, cts)`
  vs a hand-computed reference. `rand`-delta==0.
- **Unit throttle drop:** a node overhears 7 distinct-ctr_lo RTSes from X with no CTSes → `apparent=7 > 6` → the next
  RTS from X addressed to us is silent-dropped (`rts_drop_originator_throttle`, NO cts_tx, NO nack_tx). A FORWARDER X
  (RTS≈CTS) stays under threshold → CTSes normally.
- **Unit airtime backstop:** drive `total_air > floor(0.25 × budget)` with high-airtime events → drop even when
  `apparent ≤ 6`.
- **Differential (band-0):** the existing gates already exercise cooperative traffic; assert
  `rts_drop_originator_throttle` count == 0 on both engines (INERT). Optionally a new `r7` cooperative scenario.

## 7a. Post-review (R4.4 adversarial review, 2026-05-31: 17 found → 14 surviving)
**[HIGH] the airtime-cap fallback (P-CAP REVISED — the original spec pin was WRONG).** The first implementation
read the Lua `(duty_cycle_budget_ms or 36000)` as "use 36000 when budget is 0/unset" — but in Lua `0` is TRUTHY, so
`(0 or 36000)=0` and the `or 36000` is DEAD code (on_init always assigns a number). The Lua therefore gives `cap=0`
at `duty_cycle=0` and the `total_air > 0` backstop drops EVERY overheard RTS — and the originator drop, unlike its
sibling duty consumers (`compute_budget_tier` dv:3561, `check_duty_cycle` dv:3574), has **NO `duty_cycle<=0`
disabled-guard**. That missing guard is a genuine Lua oversight. **FIX (in BOTH engines, per the both-engines rule):**
gate the airtime BACKSTOP on `budget > 0` (with no budget there is no airtime share to enforce); the COUNT threshold
(`apparent > max`) stays always-active (budget-independent). C++ `node_mac.cpp` + Lua `dv:9995`. This keeps the
duty-disabled unit-test nodes from spuriously dropping (the original symptom: 8 tests failed when budget=0 → cap=0),
is differential-faithful, and removes the `36000` substitution. Regression test added (duty=0: a legit RTS CTSes;
a count-flooder still drops). **Differential-neutral:** every gate node carries `_sim_duty_cycle=0.01` → budget 36000.
- **[HIGH] #02 src_hint int8_t→int16_t.** `RxMeta.src_hint` was `int8_t`; the new `src_hint >= 0` guard would
  misread a node_id ≥ 128 (cast to negative) as "no source" and skip track+drop (and 255 aliased the −1 sentinel).
  Widened to `int16_t` (hal.h + FirmwareNode); the `uint8_t` casts still round-trip. Latent (scenarios use small ids).
- **[LOW] fixed:** #04 the misleading cap comment; #09 the drop event `from` field (r.src→`meta.src_hint`, Lua
  `meta.src`); #13 added `threshold_airtime_ms`+`window_ms` to the drop event (9 fields, Lua parity).
- **Verified faithful (no fix):** the prune/dedup-first/insertion-order/no-air-re-add (#06), draw-freeness +
  ordering-neutrality (#12), and gate-inertness (#10 — r5_cascade's same-flight retries reuse `ctr_lo` so the
  distinct count stays ≤3, well under 6).
- **Deferred:** #07 (track raw `ctr_lo` vs compute masked — equal since `ctr_lo` is 4-bit; the mask is defensive);
  #08 (the `rts_relay_exempt` emit — dormant until the R7 gateway relay path).
