# R4.5 — LBT (listen-before-talk) + on_radio_busy — design spec (the LAST R4 slice)

**Status:** DESIGN (grounded; the determinism-critical checkpoint — recommend a fresh focused effort). **Date:**
2026-05-31. **Scope:** R4.5 — wrap the TX path in listen-before-talk: defer an initiating TX (RTS/NACK) or a flood
TX (beacon) when the channel is busy; the `on_radio_busy` stash/retry loop. **Adds 2–4 new mt19937 draws — the
HIGHEST determinism risk of R4.** Completes R4. **Source of truth:** `dv_dual_sf.lua`. Delicate firmware (the TX
path, now split across node_mac.cpp/node_mac_rx.cpp) → INLINE. USER commits.

## 1. The mechanism (Lua)
- **`tx_initiating(bytes, opts, after_tx)`** (dv:3680-3718) — wraps EVERY initiating TX (RTS/NACK). LBT pre-check:
  `if lbt_enabled && !opts.__lbt_done && channel_busy_until() > now` → `wait = busy_until - now`; `delay = wait +
  rand(0, lbt_backoff_ms+1)` [**DRAW**]; set `__lbt_done` (draw ONCE); emit `tx_lbt_defer kind=initiating`; re-call
  after `delay`; return. Plus a `__pending_tx_ref` staleness guard: a deferred RTS cancels (`rts_tx_cancelled_stale`)
  if `pending_tx` changed while it waited. Then `tx_with_retry`.
- **`tx_flood(bytes, opts)`** (dv:3765-3814) — wraps FLOOD TX (beacons). (a) duty-cycle pre-check
  (`check_duty_cycle` → `duty_cycle_blocked` skip); (b) LBT: `wait = busy_until - now`; if `wait >
  flood_lbt_max_defer_ms` → DROP (`tx_flood_skipped`); else if `wait > 0` → `delay = wait + rand(0, lbt_backoff_ms+1)`
  [**DRAW**] + `tx_lbt_defer kind=flood` + defer; else TX now.
- **`on_radio_busy(info)`** (the C++ NO-OP stub, node.cpp ~152) — the stash/retry loop: `TX_DEFER_MAX_RETRIES=3`
  re-attempts at `wait + 2 + rand(0, lbt_backoff_ms+1)` [**DRAW**], with the DATA-M re-emit path. Consumes the
  `BusyInfo{reason, tag, sf, busy_until_ms}` the radio HAL already defines (hal.h:37).
- **Knobs** (NodeConfig + the host): `lbt_enabled` (default true, but the sim host knob `INode::lbtEnabled()` already
  gates the sim-side LBT defer — R3.x); `lbt_backoff_ms`; `flood_lbt_max_defer_ms`. The `channel_busy_until()` Hal
  primitive exists (wired + inert at R3.x).

## 2. DETERMINISM (the crux — same gate-inertness pattern as R4.3)
New draws fire ONLY when `lbt_enabled && channel_busy_until() > now`. **EVERY current gate sets `lbt_enabled=false`**
(r3/r4_forced/r5_cascade/hop_budget_chain/r6 all carry `"lbt_enabled": false`) → the LBT pre-check is skipped → NO
draw → R4.5 is byte-identical on every gate. The draws fire only in an `lbt_enabled=true` scenario, on BOTH engines,
so they stay aligned IF the C++ matches the Lua draw-for-draw: **strict call-site order `tx_initiating` <
`tx_flood` < `on_radio_busy`**, each `rand(0, lbt_backoff_ms+1)`, each behind its exact gate (`lbt_enabled` +
`busy_until > now`; the `__lbt_done` once-guard so a deferred re-call doesn't re-draw). Lock with a rand-order golden
(zero draws when `lbt_enabled=false`; one per deferral when enabled; the once-guard).

## 3. Recommended split (grounding)
- **R4.5a — the LBT PRE-CHECKS** (`tx_initiating` + `tx_flood`), `on_radio_busy` stays stubbed. 2 draw sites.
  The invasive part: a `Node::tx_initiating(bytes, len, TxOpts&)` wrapper that the RTS (`tx_rts_retry`) + NACK emits
  route through instead of calling `_hal.tx` directly; a `tx_flood` for `emit_beacon`. A new `kLbtDeferTimerId` for the
  deferred re-call + the `__lbt_done`/`__pending_tx_ref` deferred state on the flight. Gate it with `lbt_enabled=false`
  (inert) + a new `lbt_enabled=true` differential.
- **R4.5b — `on_radio_busy`** stash/retry loop (un-stub). +1-2 draws (the retry + the DATA-M re-emit). A
  self-tx-in-flight / half-duplex scenario fires it. Establish R4.5a's 2 sites determinism-tested in isolation FIRST.

## 4. Pins (to decide at implementation)
| # | Decision | Lean |
|---|----------|------|
| **P-WRAP** | how the LBT wraps the TX path | a `Node::tx_initiating(...)` method each initiating TX site calls (RTS/NACK), mirroring the Lua wrapper — NOT intercepting at `_hal.tx` (the Lua wraps per-site, and the `__pending_tx_ref`/`after_tx` semantics are per-call). |
| **P-SPLIT** | R4.5a then R4.5b, or both | R4.5a first (2 sites, isolate determinism), then R4.5b (the grounding's strong rec — adding R4.5b before R4.5a re-aligns the same draws twice). |
| **P-DEFER-STATE** | `__lbt_done` / `__pending_tx_ref` | per-flight fields on the deferred item (the C++ timer-id model needs a stash, like the BUSY_RX nack_wait). `__lbt_done` is the draw-once guard; `__pending_tx_ref` cancels a stale deferred RTS. |
| **P-CONFIG** | `lbt_backoff_ms` / `flood_lbt_max_defer_ms` | NodeConfig fields (Lua reads `config.X or <default>`); FirmwareNode plumb; defaults from dv:8628+. |

## 5. Gate / tests
- **Rand-order golden:** `lbt_enabled=false` → a TX draws ZERO LBT rand (the gate invariant); `lbt_enabled=true` +
  a busy channel (`channel_busy_until > now`) → ONE draw per deferred TX, at the `tx_initiating`-before-`tx_flood`
  order; the `__lbt_done` once-guard (a deferred re-call draws nothing more).
- **Unit:** `tx_initiating` defers + re-fires; the `__pending_tx_ref` staleness cancel; `tx_flood` drop when
  `wait > flood_lbt_max_defer_ms`; `on_radio_busy` retry cap (3) then giveup.
- **Differential:** the 5 gates stay band-0 (`lbt_enabled=false` → inert). A NEW `lbt_enabled=true` scenario with a
  forced busy channel is the first LBT-band differential (REPORT the funnel; assert the delivery band).

## 6. Non-goals
The load-adaptive requeue gate (separate, draw-free — can fold here or stay deferred); the schedule-TX fixup
(`apply_schedule_tx_fixup`, the R7 gateway-schedule plane); the DATA-M re-emit beyond the basic on_radio_busy path.
